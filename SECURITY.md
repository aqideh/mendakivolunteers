# Security policy

Do not publish suspected vulnerabilities, credentials, volunteer data, or exploit details in a public issue.

Use a private GitHub Security Advisory for this repository where available, or contact the repository owner through an approved MENDAKI channel. Include the affected commit, reproduction conditions, likely impact, and any evidence that does not contain personal data.

No production secrets should ever be committed. Rotate a credential immediately if it may have entered source control, logs, a pull request, or an untrusted development environment.
